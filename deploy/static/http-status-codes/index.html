HTTP Status Codes

Reference the most common HTTP response codes with short explanations for debugging requests, documenting APIs and reviewing server responses.

Use the jump links to move between response classes and keep a quick lookup nearby while working with APIs.

1xx 2xx 3xx 4xx 5xx

HTTP response code table

Code Name Meaning

1xx Informational

Informational responses indicate that the request was received and processing can continue.

2xx Success

Success responses indicate the request was received, understood and accepted.

3xx Redirection

Redirection responses tell the client that additional action is needed to complete the request.

4xx Client Errors

Client error responses indicate a problem with the request itself, authentication or permissions.

5xx Server Errors

Server error responses indicate the server failed while trying to handle an apparently valid request.

Examples

200 OK
404 Not Found
503 Service Unavailable

FAQ

What is HTTP 404?

HTTP 404 means the requested resource could not be found at the specified URL.

What is the difference between 301 and 302?

A 301 redirect is intended as permanent, while 302 usually signals a temporary move.

What does 503 mean?

HTTP 503 means the service is temporarily unavailable, often because of maintenance or overload.
